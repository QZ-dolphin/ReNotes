### 报错：. : 无法加载文件 C:\Users\Hidone\Documents\WindowsPowerShell\profile.ps1，因为在此系统上禁止运行脚本。有关详细信息，请参阅 https:/go.microsoft.com/fwlink/?LinkID=135170 中的 about_Execution_Policies。 
**出现情况：** 第一次运行vscode控制台时
**原因：** 出现此错误是因为Windows PowerShell默认不允许执行未经签名的脚本。要解决这个问题，需要更改PowerShell的执行策略：
- 步骤1：在 Windows 搜索栏中输入“PowerShell”；右键单击 Windows PowerShell，选择“以管理员身份运行”。
- 步骤2：在 PowerShell 中输入命令 Get-ExecutionPolicy 来查看当前的执行策略
- 步骤3：若要允许脚本运行，可以将执行策略更改为 RemoteSigned 或 Unrestricted。
   ```powershell
   Set-ExecutionPolicy RemoteSigned
   ``` 
   系统会提示确认更改策略，输入 Y 并按回车键确认。

   执行策略选项的含义：
  - Restricted：默认设置。不允许任何脚本运行。
  - RemoteSigned：允许本地脚本运行，但从网络下载的脚本必须有签名。
  - Unrestricted：允许所有脚本运行，但运行风险较高。
- 步骤4：重启PowerShell以确保设置生效。

### vscode执行python程序的时候，如果涉及到写相对路径，那么都是以根目录（即 你拖进vscode的文件夹）开始的，而不是py文件的所在文件夹，如何取消？
- 点击vscode左下角齿轮按钮->设置->扩展->python->勾选`execute in file dir`
